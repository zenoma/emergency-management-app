\chapter{Análisis de viabilidad}
\label{chap:viabilidade}

\lettrine{E}{n este capítulo} se analiza la viabilidad del proyecto desde tres perspectivas complementarias: dimensión temporal, dimensión tecnológica y dimensión económica.

\section{Viabilidad temporal}
Para la realización de este proyecto se ha estimado una duración aproximada de cuatro meses. Dicha planificación incluye las fases de análisis, diseño, implementación, pruebas y redacción de la memoria. Además, se contempla la familiarización con las nuevas tecnologías.

La metodología usada en la realización del proyecto es ágil, lo que permite adaptarse a cambios o imprevistos durante el desarrollo. La experiencia previa del alumno en proyectos similares y el conocimiento de las tecnologías involucradas reducen el riesgo de desviaciones significativas sobre la planificación inicial.

\section{Viabilidad tecnológica}
Para el estudio de viabilidad tecnológica se han considerado las tecnologías heredadas del trabajo previo \cite{TFGAdrianGarcia2025}, así como las incorporaciones necesarias para completar la parte móvil y el sistema de avisos.

La solución se apoya en un backend para servicios REST y lógica de negocio, integrado con una base de datos espacial que permite gestionar cuadrantes, ubicaciones y recursos georreferenciados. Estas tecnologías ya conocidas por el autor, como Spring Boot y PostgreSQL/PostGIS, ofrecen un entorno estable para el desarrollo del sistema.

La interfaz web mantiene una arquitectura orientada a la gestión operativa, mientras que la parte móvil se resuelve con una aplicación nativa pensada para acceder a sensores del dispositivo y mostrar la información de forma cómoda en pantalla. Para la parte web, el alumno también cuenta con experiencia previa en React.

En lo relativo a la aplicación móvil, se opta por Kotlin y Jetpack Compose, tecnologías ampliamente adoptadas para el desarrollo Android. Cuentan con documentación extensa y guías oficiales para su integración con backends Java/Spring. Su adopción aporta ventajas en productividad, interoperabilidad móvil y soporte nativo para notificaciones.

En conjunto, la dimensión tecnológica se apoya en tecnologías ya conocidas y probadas, con una curva de aprendizaje manejable y documentación suficiente para cubrir las integraciones necesarias.

\section{Viabilidad económica}
El proyecto no requiere una inversión económica significativa en hardware o licencias de software. El software utilizado es mayoritariamente Open Source, por lo que no hay costes de licencia en el entorno de desarrollo académico. Para el desarrollo y las pruebas, estas pueden ejecutarse con recursos disponibles, por lo que no es necesario adquirir nuevo hardware.

En el ámbito económico, el proyecto no requiere una inversión inicial y los costes asociados a servicios externos pueden gestionarse mediante planes gratuitos o alternativas \emph{open source}. 

\section{Conclusión del análisis de viabilidad}
En conjunto, el análisis muestra una planificación temporal acotada, una base tecnológica conocida y un coste económico reducido. Estos elementos sitúan el proyecto dentro de un margen de desarrollo asumible con los recursos disponibles.
