\chapter{Análisis de viabilidad}
\label{chap:viabilidade}

\lettrine{E}{n este capítulo} se analiza la viabilidad del proyecto desde tres perspectivas complementarias: dimensión temporal, dimensión tecnológica y dimensión económica. 

\section{Viabilidad temporal}
Para la realización de este proyecto se ha estimado una duración aproximada de cuatro meses. Dicha planificación incluye las fases de análisis, diseño, implementación, pruebas y redacción de la memoria. Además, se contempla la familiarización de las nuevas tecnologías. 

La metodología usada en la realización del proyecto es ágil, lo que permite adaptarse a cambios o imprevistos durante el desarrollo. Además, la experiencia previa del autor en proyectos similares y el conocimiento de las tecnologías involucradas contribuyen a una ejecución eficiente y a la minimización de retrasos. En conclusión, la dimensión temporal del proyecto es viable y se ajusta a los objetivos establecidos.

\section{Viabilidad tecnológica}
Para el estudio de viabilidad tecnológica se han considerado las tecnologías heredadas del trabajo previo \cite{TFGAdrianGarcia2025}, así como las incorporaciones necesarias para completar la parte móvil y el sistema de avisos.

La solución se apoya en un backend para servicios REST y lógica de negocio, integrado con una base de datos espacial que permita gestionar cuadrantes, ubicaciones y recursos georreferenciados. Estas tecnologías conocidas por el autor, como Spring Boot y PostgreSQL/PostGIS, ofrecen un entorno robusto y eficiente para el desarrollo del sistema.

La interfaz web mantiene una arquitectura orientada a la gestión operativa, mientras que la parte móvil se resuelve con una aplicación nativa pensada para acceder a sensores del dispositivo y mostrar la información de forma cómoda en pantalla. Para la parte web, el autor también tiene experiencia previa con React, lo que facilita la implementación de las nuevas funcionalidades. 

En lo relativo a la aplicación móvil, se opta por Kotlin y Jetpack Compose, tecnologías modernas y recomendadas para el desarrollo Android. Estas tecnologías están ampliamente adoptadas, cuentan con extensa documentación, y existen guías oficiales para su integración con backends Java/Spring. Por tanto, su adopción no supone un riesgo tecnológico significativo, aportan ventajas en productividad, interoperabilidad móvil y soporte nativo para notificaciones.

En conclusión, la dimensión tecnológica del proyecto es viable, ya que se basa en tecnologías conocidas y probadas, con una curva de aprendizaje manejable. La experiencia previa del autor en estas tecnologías y la disponibilidad de recursos y documentación garantizan que el desarrollo se pueda llevar a cabo sin contratiempos técnicos significativos.

\section{Viabilidad económica}
El proyecto no requiere una inversión económica significativa en hardware o licencias de software. El software utilizado es mayoritariamente Open Source, por lo que no hay costes de licencia en el entorno de desarrollo académico.

Para el desarrollo y las pruebas, estas pueden ejecutarse con recursos disponibles por lo que no es necesario adquirir nuevo hardware. Sobre el uso de servicios en la nube o APIs de terceros pueden surgir costes, pero se controlan gracias a los planes gratuitos o a la monitorización de uso.

En el ámbito de la viabilidad económica, el proyecto es viable, ya que no requiere una inversión inicial y se pueden gestionar los costes asociados a servicios externos mediante planes gratuitos o alternativas \emph{open source}. Además, la experiencia previa del autor en la gestión de proyectos similares contribuye a una planificación financiera eficiente y a la minimización de gastos innecesarios.

\section{Conclusión del análisis de viabilidad}
Como conclusión del análisis de viabilidad, el proyecto es completamente factible desde las perspectivas temporal, tecnológica y económica. La planificación de cuatro meses es realista y se ajusta a los objetivos establecidos, mientras que la elección de tecnologías conocidas y probadas minimiza los riesgos técnicos. Además, el uso de software Open Source y la gestión cuidadosa de los recursos garantizan que el proyecto se pueda llevar a cabo sin una inversión económica significativa. En conjunto, estos factores aseguran que el proyecto se pueda completar con éxito dentro del marco temporal y con los recursos disponibles.
