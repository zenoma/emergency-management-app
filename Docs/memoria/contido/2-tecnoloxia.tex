\chapter{Base Tecnológica}
\label{chap:basetecnoloxica}

\lettrine{E}{}n este capítulo se describen las tecnologías que conforman la base del proyecto, organizadas en dos grandes bloques. El primero recoge las tecnologías que se mantienen del trabajo previo \cite{TFGAdrianGarcia2025}, con una breve descripción de cada una, así como aquellas que se han dejado de usar. El segundo bloque agrupa las tecnologías que se incorporan como novedad en este \acrshort{tfm}, explicando su propósito y la motivación de su elección.

\section{Tecnologías de continuidad}

Se consideran tecnologías de continuidad aquellas que ya estaban presentes en el \acrshort{tfg} y que se mantienen en esta evolución del proyecto. Para cada una se incluye una descripción breve de su función.

\subsection{Lenguajes}

\subsubsection{Java}
Java \cite{java} es un lenguaje de programación orientado a objetos, multiplataforma y ampliamente utilizado en el desarrollo de aplicaciones empresariales. Se mantiene como lenguaje principal del backend por su madurez, su integración natural con Spring y su ecosistema de librerías.

\subsubsection{SQL}
SQL (Structured Query Language) \cite{sql} es el lenguaje estándar para la definición, consulta y manipulación de bases de datos relacionales. Se conserva como base para gestionar los datos persistidos de la aplicación.

\subsubsection{HTML y CSS}
HTML (HyperText Markup Language) y CSS (Cascading Style Sheets) \cite{html_css} son los lenguajes fundamentales para la estructura y presentación de páginas web. Continúan siendo la base de la interfaz web de la plataforma.

\subsubsection{JavaScript}
JavaScript \cite{javascript} es un lenguaje de programación interpretado que permite añadir interactividad a las páginas web. Se conserva en el frontend como base de React y de la lógica de interacción del navegador.

\subsubsection{JSX}
JSX (JavaScript XML) \cite{jsx} es una extensión de sintaxis para JavaScript que permite escribir código similar a HTML dentro de componentes React. Se mantiene por su claridad al combinar estructura visual y lógica.

\subsubsection{LaTeX}
LaTeX \cite{latex} es un sistema de composición de textos orientado a documentos técnicos y científicos. Se mantiene para la redacción y maquetación de esta memoria.

\subsubsection{JSON}
JSON (JavaScript Object Notation) \cite{json} es un formato ligero de intercambio de datos basado en texto. Continúa siendo el formato empleado para la comunicación entre cliente y servidor.

\subsection{Frameworks y librerías servidor}

\subsubsection{Spring}
Spring \cite{spring} es un framework de desarrollo de aplicaciones Java que proporciona un modelo integral de programación y configuración. Permanece como núcleo del backend por su robustez, su soporte de inyección de dependencias y su ecosistema de módulos.

\subsubsection{Spring Boot}
Spring Boot \cite{springboot} es una extensión de Spring que simplifica la configuración y el despliegue de aplicaciones mediante valores por defecto y auto-configuración. Sigue siendo la base sobre la que se levanta el backend de la plataforma.

\subsubsection{Spring Data}
Spring Data \cite{springdata} es un módulo de Spring que abstrae el acceso a distintas fuentes de datos. Se mantiene como capa intermedia entre la lógica de negocio y los repositorios, reduciendo la cantidad de código de persistencia.

\subsubsection{JPA}
JPA (Jakarta Persistence API) \cite{jpa} es una especificación de Java para el mapeo objeto-relacional. Continúa utilizándose para modelar la persistencia de las entidades del dominio.

\subsubsection{Hibernate}
Hibernate \cite{hibernate} es la implementación de referencia del estándar JPA. Se conserva para gestionar entidades, relaciones, consultas y carga perezosa.

\subsubsection{JDBC}
JDBC (Java Database Connectivity) \cite{jdbc} es una API estándar de Java para el acceso directo a bases de datos relacionales. Se mantiene como soporte de bajo nivel para operaciones que requieren consultas nativas.

\subsection{Frameworks y librerías web}

\subsubsection{React}
React \cite{react} es una biblioteca de JavaScript para la construcción de interfaces de usuario basadas en componentes reutilizables. Se conserva como base del frontend web por su modularidad y su ecosistema.

\subsubsection{Redux}
Redux \cite{redux} es un contenedor predecible del estado de aplicaciones JavaScript. Sigue siendo el gestor central del estado global de la aplicación web.

\subsubsection{RTK Query}
RTK Query \cite{rtkquery} es una herramienta integrada con Redux Toolkit que simplifica la carga, caché y sincronización de datos desde APIs. Se mantiene como solución principal de acceso al backend desde el frontend web.

\subsubsection{Material UI}
Material UI \cite{materialui} es una biblioteca de componentes visuales para React que implementa el diseño Material Design. Continúa proporcionando los elementos de interfaz de la aplicación web.

\subsubsection{React Map GL y MapLibre}
React Map GL \cite{reactmapgl} es un componente React para integrar mapas interactivos, y MapLibre \cite{maplibre} es una biblioteca de mapas de código abierto compatible con los formatos de Mapbox. Esta combinación se mantiene en la web para proporcionar una capa cartográfica abierta.

\subsubsection{Open Weather Map}
Open Weather Map \cite{openweathermap} es un servicio que proporciona datos meteorológicos actualizados. Se mantiene como fuente externa de información climática relevante en contextos de emergencia.

\subsubsection{Mapbox}
\label{sec:mapbox}
Mapbox \cite{mapbox} es una plataforma de mapas y servicios de localización. En el \acrshort{tfg} se utilizaba como proveedor cartográfico principal, pero en este trabajo se deja de usar para reducir la dependencia de claves API y de planes de uso restringidos. La visualización geográfica se apoya ahora en MapLibre y MapTiler \ref{sec:maptiler}, que encajan mejor con el enfoque abierto del proyecto.

\subsection{Pruebas}

\subsubsection{JUnit}
JUnit \cite{junit} es un framework de pruebas unitarias para Java. Se conserva como herramienta principal para la verificación de la lógica del backend.

\subsection{Protocolos}

\subsubsection{HTTP/S}
HTTP (Hypertext Transfer Protocol) y HTTPS (HTTP Secure) \cite{http} son los protocolos de comunicación empleados en la transferencia de datos entre cliente y servidor. Continúan siendo la base de la comunicación de la plataforma.

\subsubsection{REST}
REST (Representational State Transfer) \cite{rest} es un estilo arquitectónico para diseñar servicios web que exponen recursos mediante operaciones bien definidas. Sigue siendo el estilo principal para la API de la plataforma.

\subsection{Herramientas de desarrollo}

\subsubsection{IntelliJ IDEA}
IntelliJ IDEA \cite{intellij} es un entorno de desarrollo integrado especializado en Java y tecnologías \acrshort{jvm}. Se mantiene como herramienta principal para el desarrollo del backend.

\subsubsection{Postman}
Postman \cite{postman} es una herramienta para probar y documentar APIs. Se conserva para la exploración y verificación de los endpoints del backend.

\subsubsection{Visual Studio Code}
Visual Studio Code \cite{vscode} es un editor de código ligero y extensible. Se mantiene como editor de apoyo para el frontend web y la documentación.

\subsubsection{DBeaver}
DBeaver \cite{dbeaver} es una herramienta de administración y consulta de bases de datos. Continúa empleándose para interactuar con PostgreSQL y PostGIS durante el desarrollo.

\subsubsection{Git}
Git \cite{git} es un sistema de control de versiones distribuido. Se mantiene para la gestión del código fuente del proyecto.

\subsubsection{Apache Maven}
Apache Maven \cite{maven} es una herramienta de automatización de compilación y gestión de dependencias para proyectos Java. Sigue siendo la base del ciclo de construcción del backend y soporta la generación de código a partir de OpenAPI.

\subsection{Sistemas de gestión de base de datos}

\subsubsection{PostgreSQL}
\label{sec:postgreSQL}
PostgreSQL \cite{postgresql} es un sistema de gestión de bases de datos relacional de código abierto, conocido por su estabilidad y extensibilidad. Se mantiene como base de datos principal del proyecto.

\subsubsection{PostGIS}
\label{sec:postGIS}
PostGIS \cite{postgis} es una extensión de PostgreSQL \ref{sec:postgreSQL} que añade soporte para datos geoespaciales. Continúa siendo esencial para las operaciones cartográficas de la plataforma.

\subsubsection{H2}
H2 \cite{h2} es una base de datos relacional ligera embebida que se empleaba en el \acrshort{tfg} para pruebas rápidas y desarrollo local. En este trabajo se elimina porque se ha optado por trabajar directamente sobre PostgreSQL \ref{sec:postgreSQL} en todos los entornos, utilizando Testcontainers \ref{sec:Testcontainers} para las pruebas de integración. Esto evita las diferencias de comportamiento entre bases de datos y proporciona un entorno de prueba más representativo.

\subsection{Tecnologías de alojamiento y construcción}

\subsubsection{Tomcat}
Tomcat \cite{tomcat} es un contenedor de servlets que permite ejecutar aplicaciones Java web. Se mantiene como contenedor de ejecución del backend.

\subsubsection{Vite}
Vite \cite{vite} es un empaquetador y servidor de desarrollo para aplicaciones web modernas. Continúa formando parte de la cadena de construcción del frontend web.

\subsubsection{Serve}
Serve \cite{serve} es una herramienta para servir ficheros estáticos de forma sencilla. Se conserva para servir la versión compilada del frontend durante pruebas o despliegues locales.

\section{Nuevas tecnologías}

Este apartado recoge las tecnologías que se incorporan por primera vez en este \acrshort{tfm}. Para cada una se explica qué es y por qué se ha incorporado.

\subsection{OpenAPI y enfoque API-first}
OpenAPI \cite{OpenAPI} es una especificación estándar para describir APIs REST mediante un documento YAML o JSON. Su incorporación responde al enfoque \textit{API-first}, que consiste en diseñar el contrato de la API antes de escribir la implementación. En este proyecto, \cite{OpenAPI} se utiliza para definir los endpoints, parámetros, respuestas y estructuras de datos en un único archivo, a partir del cual se generan automáticamente los controladores y DTOs del backend. Este enfoque garantiza que la API esté bien documentada y que todas las capas (backend, frontend web y app móvil) consuman una interfaz coherente, evitando desajustes entre lo que el servidor expone y lo que los clientes esperan.

\subsection{Kotlin}
Kotlin \cite{kotlin} es un lenguaje de programación moderno que se ejecuta sobre la \acrfull{jvm} y es plenamente interoperable con Java. Se incorpora como lenguaje principal de la aplicación Android por su concisión, su seguridad frente a nulos y su fuerte integración con el ecosistema Android. Kotlin permite reducir código redundante y facilita el mantenimiento de la aplicación móvil, a la vez que ofrece compatibilidad total con librerías Java existentes.

\subsection{Gradle}
Gradle \cite{gradle} es un sistema de automatización de compilación que combina la flexibilidad de Ant con las convenciones de Maven, utilizado ampliamente en proyectos Android. Se añade como gestor de dependencias y construcción de la app móvil, ya que permite definir módulos, variantes de compilación y tareas personalizadas de forma declarativa.

\subsection{Jetpack Compose}
Jetpack Compose \cite{jetpack_compose} es un toolkit moderno de Android para construir interfaces de usuario de forma declarativa con código Kotlin. Se incorpora como reemplazo del sistema tradicional basado en XML, ofreciendo una forma más intuitiva y reactiva de crear pantallas. Esto es especialmente útil en una aplicación con múltiples vistas y navegación compleja, ya que reduce la cantidad de código necesario y facilita la gestión del estado de la interfaz.

\subsection{Material Design}
Material Design \cite{material_design} es un sistema de diseño creado por Google que define principios visuales, componentes y patrones de interacción para aplicaciones móviles. Se adopta como guía de estilo de la app Android para ofrecer una experiencia de usuario coherente, accesible y reconocible, siguiendo los estándares actuales de la plataforma.

\subsection{MapTiler}
\label{sec:maptiler}
MapTiler \cite{maptiler} es un proveedor de estilos cartográficos y teselas de mapa. Se incorpora como fuente de estilos visuales para los mapas web, en sustitución de Mapbox \ref{sec:mapbox}, ya que ofrece un modelo de uso más flexible y se alinea con el enfoque abierto del proyecto.

\subsection{HttpClient}
HttpClient \cite{httpclient} es una biblioteca de Kotlin/Java para realizar peticiones HTTP de forma sencilla y eficiente. Se añade en la app Android como capa de comunicación con el backend, centralizando el acceso a los servicios REST de la plataforma.

\subsection{Firebase Cloud Messaging}
Firebase Cloud Messaging (FCM) \cite{firebase} es un servicio de Google que permite el envío de notificaciones push a dispositivos móviles. Se incorpora para mantener informados a los usuarios en tiempo real sobre emergencias, cambios en asignaciones o alertas relevantes, sin necesidad de que la aplicación esté abierta.

\subsection{Firebase Messaging Service}
Firebase Messaging Service \cite{firebase_messaging_service} se añade para gestionar la recepción de mensajes procedentes de FCM y reaccionar a notificaciones o eventos remotos. Esta tecnología permite que la aplicación móvil pueda recibir y procesar notificaciones push incluso cuando no está en primer plano, lo que es crucial para garantizar que los usuarios reciban información importante de manera oportuna. FMS se encarga de manejar la lógica de recepción de mensajes, permitiendo mostrar notificaciones, actualizar la interfaz de usuario o realizar acciones específicas en función del contenido del mensaje recibido.

\subsection{Testcontainers}
\label{sec:testcontainers}
Testcontainers \cite{testcontainers} es una biblioteca Java que permite levantar contenedores Docker efímeros para su uso en pruebas. Se incorpora para ejecutar las pruebas de integración del backend sobre instancias reales de PostgreSQL \ref{sec:postgreSQL} con PostGIS \ref{sec:postGIS}, eliminando las diferencias que existían al usar H2 en el \acrshort{tfg}. Al estar basado en Docker, garantiza que el entorno de pruebas sea idéntico al de producción, lo que mejora la fiabilidad de las pruebas y reduce problemas derivados de diferencias entre bases de datos.
